# Sample Requirements — E-Commerce Platform (Backwards Compatibility Demo)

## Functional Requirements

- Req-1: Users must be able to register with email and password.
- Req-2: Users must be able to log in with Google OAuth.
- Req-3: The product catalog must support filtering by category, price range, and rating.
- Req-4: Users must be able to add items to a shopping cart and modify quantities.
- Req-5: Users must be able to checkout using credit card or PayPal.

## Non-Functional Requirements

- Req-6: The product catalog page must load in under 2 seconds.
- Req-7: The system must support at least 1000 concurrent users without degradation.
- Req-8: All user passwords must be stored using bcrypt with salt.
